## 阶段 7 思维导图修复详情

### 1. 节点大小动态自适应
- 新增 `estimateTextLines(text, fontSize, maxWidth)` 函数
  - 中文字符 ≈ fontSize × 1.05，拉丁字符 ≈ fontSize × 0.58
- `nodeH` 从固定值 `82 | 52` → 动态计算：`名称行数×17 + 公式行数×17 + 描述行数×13 + 14`，最小 52px
- 移除所有 `overflow:hidden` + `text-overflow:ellipsis` + `white-space:nowrap` 截断
- 改为 `word-wrap:break-word; overflow-wrap:break-word` 自动换行
- formula 错误回退移除 `substring(0,40)` → 显示完整公式
- 描述移除 `substring(0,50)` → 显示完整描述

### 2. 连线箭头修复（两个根因）
- **根因 1**：`svg.innerHTML` 创建的 `<marker>` 不在 SVG 命名空间，导致 `url(#arrowhead)` 引用失效
- **修复 1**：改用 `createElementNS('http://www.w3.org/2000/svg', ...)` 创建 defs/marker/polygon/circle
- **根因 2**：路径端点位于节点中心，箭头被节点矩形完全遮盖
- **修复 2**：新增 `clipToRectEdge()` 函数，将路径起点/终点从节点中心裁剪到矩形边缘（+2px 间距）
- `calcCurvePath` 改为读取 `node._nodeW` / `node._nodeH` 进行裁剪
- `renderMindmap` 新增预计算阶段：先算所有节点尺寸存入 `node._nodeW/H`，再画连线和节点
- 箭头三角形：markerWidth=14, markerHeight=10
- 起点圆点标记 `<marker id="startdot">` — 连线起点显示蓝色小圆点
- path 设置 `marker-start="url(#startdot)"` + `marker-end="url(#arrowhead)"`

### 3. Ctrl+Z 撤销功能
- `mindmapState` 新增 `undoStack: []`（最多 50 条快照）
- `pushUndoState(course)` — 深拷贝 connections / nodePositions / nodeColors 入栈
- `performUndo(course)` — 出栈恢复 + saveMindmapData + 重绘 + Toast 提示
- 所有破坏性操作前自动保存快照：拖拽节点、隐藏节点、新建/删除连线、更改颜色
- Ctrl+Z / Cmd+Z 触发撤销，`e.preventDefault()` 防止浏览器默认行为

### 4. 连线选中与删除
- 隐形 14px 宽点击区域（`stroke:transparent`）附在每条连线上
- 点击连线 → 选中（红色 stroke + 加粗 + drop-shadow 发光，标签同步变红加粗）
- 再次点击已选中连线 → 取消选中
- 点击 SVG 空白区域 → 取消选中
- Delete / Backspace 键删除选中连线（带 pushUndoState 撤销支持）
- ESC 键同时支持取消连线模式和取消选中

### 5. Bugfix：死代码导致思维导图不显示
- **现象**：页面显示「加载中」，思维导图完全空白
- **原因**：`calcCurvePath` 替换时残留旧函数体 8 行（`const dx = ...` 等），出现在新 `return` 之后 → JS 解析失败
- **修复**：删除残留死代码
